# Skills 参考说明

本项目将“技能（Skills）”理解为：AI 在生成报告时可调用的一组本地能力与对应使用规范。

## 1. `text_reader`

- **用途**：读取 `.txt`、`.md` 等纯文本资料。
- **输入**：文件路径。
- **输出**：文本内容、编码信息、告警。
- **适用场景**：访谈纪要、会议记录、说明文档、网页摘录。
- **注意事项**：若出现乱码，优先尝试重新导出 UTF-8 文件。

## 2. `pdf_reader`

- **用途**：读取 PDF 文本内容。
- **输入**：文件路径。
- **输出**：按页拼接的文本、页数、告警。
- **适用场景**：研究报告、制度文件、合同、公告、说明书。
- **注意事项**：扫描版 PDF 可能无法直接提取文本，应标记“待 OCR”。

## 3. `docx_reader`

- **用途**：读取 Word 文档段落与表格文本。
- **输入**：文件路径。
- **输出**：段落文本、表格文本、段落数、表格数。
- **适用场景**：方案、汇报稿、会议纪要、制度草案。
- **注意事项**：复杂页眉页脚、批注和修订痕迹当前不做完整解析。

## 4. `pptx_reader`

- **用途**：读取 PowerPoint 幻灯片中的文本。
- **输入**：文件路径。
- **输出**：按页组织的幻灯片文本、页数。
- **适用场景**：汇报材料、商业计划、培训课件。
- **注意事项**：图片中的文字不一定可提取，应在报告中说明可能遗漏视觉内容。

## 5. `xlsx_reader`

- **用途**：读取 Excel 工作表中的文本与数值。
- **输入**：文件路径。
- **输出**：按工作表拼接的表格文本、工作表名称。
- **适用场景**：经营数据、调研表、预算表、清单表。
- **注意事项**：合并单元格和公式结果依赖文件保存状态，必要时需人工复核。

## 6. `web_search`

- **用途**：在用户明确允许且环境已启用时，抓取公开搜索结果页中的候选链接。
- **输入**：查询词、结果数量上限。
- **输出**：标题、URL、来源搜索页、告警。
- **适用场景**：补充查找最新政策动态、官网公告、标准发布页、公开行业资料。
- **注意事项**：
  - 默认关闭，只有显式设置 `REPORT_GENERATOR_ENABLE_WEB=1` 后才可用。
  - 当前采用免费网页搜索抓取方案，不保证搜索结果页结构长期稳定。
  - 联网结果只能作为补充线索，不能直接替代本地原始资料。
  - 写入报告前，必须回到政府官网、标准原文、权威数据库或正式发布页面进行二次核验。

## 7. `fetch_web_page`

- **用途**：抓取指定网页正文，供 AI 进一步阅读和核验。
- **输入**：网页 URL。
- **输出**：网页标题、正文文本、告警。
- **适用场景**：阅读政府官网公告页、标准发布页、监管机构通知页、项目介绍页。
- **注意事项**：
  - 默认关闭，只有显式设置 `REPORT_GENERATOR_ENABLE_WEB=1` 后才可用。
  - 页面正文抽取可能包含导航、页脚、推荐阅读等噪声。
  - 正文内容仍需人工或规则进一步甄别，尤其是发布日期、发文机关、附件链接。

# AI 调用约定

当你把本项目产出的 `outputs/ai_prompt_package.md` 交给 AI 时，AI 应按以下约定工作：

1. **先看资料清单**：确认哪些文件已成功读取，哪些失败。
2. **优先使用证据摘录**：先基于证据摘录形成初步判断。
3. **必要时回看来源全文**：遇到关键结论时，回到来源文件做二次核对。
4. **写作必须带依据**：所有关键结论尽量绑定来源文件名。
5. **证据不足要显式说明**：不得因为报告需要完整而编造事实。
6. **联网检索要后置使用**：只有在本地资料不足、且用户允许联网时，才可调用 `web_search` 补充线索。
7. **联网结果必须复核**：对关键数字、政策名称、发布日期、机构名称，必须用原始权威来源再次确认。
8. **正文抓取用于核验**：拿到候选链接后，应优先用 `fetch_web_page` 抽取正文，再判断是否采用。

## 推荐调用顺序

1. 调用目录扫描逻辑，枚举所有资料。
2. 按扩展名自动路由到相应读取 skill。
3. 汇总为证据包。
4. 根据报告主题筛选高相关证据。
5. 如本地证据不足且用户允许联网，再调用 `web_search` 补充公开线索。
6. 对候选链接调用 `fetch_web_page` 抽取正文。
7. 对联网结果做二次核验。
8. 生成大纲。
9. 按“结论-依据-风险-建议”输出正文。
10. 使用质量检查清单进行自检。

## 标准输出要求

- 标注引用来源文件名。
- 区分“事实”“判断”“推断”。
- 对缺失信息建立“待确认项”列表。
- 若多个来源冲突，列出冲突点与可能原因。
- 若使用了联网信息，必须标注检索时间与最终采用的权威来源。
